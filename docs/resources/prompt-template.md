# Prompt 範本

不需要把 prompt 寫成長篇規格，但要讓目標、素材、限制與驗收方式可以被辨認。

## 通用格式

```md
背景：誰會使用，現在遇到什麼問題。
目標：完成後應該產生什麼改變。
素材：相關檔案、截圖、網址或資料。
限制：不能改什麼、需要保留什麼。
驗收：要執行哪些測試，或看到哪些結果。
交付：是否需要 branch、commit、PR 或部署。
```

## 使用方式

先寫能影響結果的資訊。Codex 可從工作區發現的細節，不必全部手動抄進 prompt。
