# Part 1｜理解專案

負責人：Benson

本頁聚焦 read-only 探索。目標不是讀完所有檔案，而是建立足以支持下一步修改的專案理解。

## 課程重點

- 建立 Repository 閱讀順序
- 找出入口、主要模組與資料流
- 以檔案位置支持重要結論
- 區分已確認內容、推測與未知事項
- 整理 Part 2 可直接使用的修改入口與驗證建議

## Demo 流程

<!-- Benson：在此補上使用的 Repository、任務情境、Prompt 與預期畫面。 -->

1. 盤點專案規則、設定、入口與測試。
2. 建立 Codebase map 並追蹤指定資料流。
3. 列出可能的修改位置、風險與待確認事項。
4. 將結果交給 Part 2。

## 交付給 Part 2

<!-- 共同確認此處的輸出格式，讓 Roy 可以直接承接。 -->

- 系統與模組摘要
- 主要資料流與檔案證據
- 建議修改位置
- 風險、未知事項與驗證建議
