# Part 2｜動手修改

負責人：Roy

本頁承接 Part 1 的 Codebase map 與修改建議，聚焦如何讓 Codex 在既有專案規範與權限邊界內完成變更。

## 課程重點

- 將理解結果轉成修改計畫
- 使用 `AGENTS.md`、Config 或 Skill 約束執行方式
- 進行最小且可審查的程式變更
- 執行測試、檢查 Diff 並整理驗證證據
- 說明各種 Codex 進階能力在工作流中的角色

## Demo 流程

<!-- Roy：在此補上修改需求、使用的 Skills／能力、驗證命令與預期畫面。 -->

1. 讀取 Part 1 的結論並確認修改範圍。
2. 提出實作與驗證計畫。
3. 完成小型修改並執行必要檢查。
4. Review Diff，說明結果、風險與後續建議。

## 完成標準

- 修改範圍與 Part 1 的證據一致
- 沒有重新進行無目的的全專案探索
- 測試或驗證命令成功
- 最終 Diff、限制與剩餘風險清楚可見
