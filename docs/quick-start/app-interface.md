# Codex App 介面

Codex App 把對話、工作區與版本差異放在同一個任務裡。你不需要逐步下指令，但需要知道目前修改在哪個分支、有哪些檔案被改動，以及驗證結果是否可信。

## 任務與工作區

每個任務應對應一個清楚的目標。工作區保存專案內容；建立分支後，修改可以和 `main` 隔離，方便檢查與回復。

## 終端與執行結果

終端顯示建置、測試與工具輸出。錯誤訊息不是額外負擔，而是 Codex 判斷下一步的重要證據。

## 差異檢視

交付前先看 diff：確認沒有意外刪除、敏感資訊或無關格式化，再決定是否 commit 與建立 PR。
