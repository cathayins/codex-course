# ChatGPT vs Codex：回答，還是交辦？

<p class="lesson-lead">ChatGPT 很像一位隨時可以討論的顧問。Codex 更像一位能進入工作環境的同事：它不只提出做法，也能打開資料夾、讀取檔案、使用工具，把結果做進真正的成品裡。</p>

## 顧問給你方向，同事把工作往前推

<div class="story-contrast story-contrast--projector">
  <section>
    <span>CHATGPT</span>
    <h3>先一起想清楚</h3>
    <p>你提出問題，它幫你整理資訊、比較選項、發想或寫出第一版內容。</p>
    <b>適合：快速問答、腦力激盪、單次分析</b>
  </section>
  <section>
    <span>CODEX</span>
    <h3>進入工作現場完成任務</h3>
    <p>你指定資料夾與目標，它讀取多個檔案、使用工具、建立或修改成果，再回來說明做了什麼。</p>
    <b>適合：多檔案、要落地、需要持續修改的工作</b>
  </section>
</div>

這不是兩條硬切開的能力界線。現在的 ChatGPT Work 也能處理檔案並製作成品；Codex 也會先和你討論。這裡要抓的是一個更實用的判斷：**這次只需要一個回答，還是要讓 AI 進入工作環境，把事情做進檔案裡？**

## 要讓 Codex 接手，先給它四件事

<div class="brief-grid">
  <section><span>01</span><b>這次要完成什麼</b><p>不要只說「幫我看一下」，說清楚最後要得到什麼。</p></section>
  <section><span>02</span><b>工作資料在哪裡</b><p>指定資料夾、檔案、選取內容，或必要的外部來源。</p></section>
  <section><span>03</span><b>哪些事情不能碰</b><p>例如不要改原始檔、不要寄出、不要連外，或先不要執行。</p></section>
  <section><span>04</span><b>回來時要帶什麼</b><p>一份整理後的檔案、一個可執行版本，或一段清楚的說明。</p></section>
</div>

這四件事不用一次寫成很長的規格。只要先交代會影響結果的資訊，就比研究一串「神奇 Prompt」更有用。
