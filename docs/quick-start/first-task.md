# 完成第一個任務

好的第一個任務不必很大，但要有真實產出。你可以請 Codex 修正文案、調整一個元件、補上測試，或整理一份可直接使用的文件。

## 描述需求

說明使用情境、目標使用者、現況問題與不可接受的結果。若有截圖或參考網站，指出你要的是資訊層級、互動方式還是視覺語言。

## 讓 Codex 執行

允許 Codex先讀取專案、提出合理假設並持續驗證。需要外部寫入或重大方向選擇時，再保留人工確認。

## 檢查過程

進度更新應讓你知道正在處理什麼、已確認什麼，以及是否遇到會改變結果的限制。
