# 認識 Codex

Codex 是可以在真實工作區讀取檔案、修改程式、執行指令並驗證成果的程式設計代理。好的使用方式不是只取得一段答案，而是把目標、限制與驗收條件交代清楚，讓工作形成可檢查的閉環。

## 適合交給 Codex 的工作

- 理解陌生專案與追查問題來源
- 實作功能、重構與補上測試
- 整理文件、資料與可重複流程
- 準備 commit、Pull Request 與交付摘要

## 人仍然負責什麼

你仍需決定真正的目標、風險邊界與完成標準。當任務會影響正式環境、敏感資料或外部使用者時，也應明確保留審批點。

## 一個好任務的形狀

一個可執行的任務通常包含：背景、要改變的結果、不可破壞的限制、可用素材，以及完成後如何驗收。
