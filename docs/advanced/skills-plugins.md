# Skills 與 Plugins

Skill 保存特定任務的操作方法與品質標準；Plugin 則能把 skills、工具與連線能力組合成一個可安裝套件。

## 何時使用 Skill

當一類任務會反覆出現，而且需要固定的檢查順序、模板或專業限制時，就適合建立 skill。

## 何時需要 Plugin

當能力不只有文字規則，還需要外部工具、資料來源或多個協作元件時，再考慮 plugin。

## 好的封裝原則

只封裝穩定且可重複的流程；把專案一次性的細節留在任務或 `AGENTS.md`。
