# 團隊工作流

團隊導入 Codex 的關鍵不是讓每個人輸入相同 prompt，而是建立共同的任務邊界、驗收方式與版本流程。

## 任務分工

把可獨立驗證的工作拆成清楚任務，避免多人同時修改同一組核心檔案。需要平行處理時，先界定輸入與交付格式。

## 審查品質

PR 應說明目的、主要取捨、驗證結果與剩餘風險。審查者聚焦行為與維護性，而不只檢查程式是否由 AI 產生。

## 累積組織知識

反覆出現的規則寫進 `AGENTS.md`；成熟的專業流程再整理成 skill，減少知識只存在個人對話裡。
