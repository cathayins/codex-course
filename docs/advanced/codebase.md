# 理解陌生 Codebase

面對陌生專案時，先找入口、資料流、共用規範與驗證命令。過早修改最容易產生局部正確、整體衝突的結果。

## 建立最小系統地圖

確認框架、建置方式、路由、狀態來源、主要元件與部署流程。這些關係足以支撐第一輪判斷，不必一開始讀完整個 repository。

## 搜尋與追蹤

從畫面文字、函式名稱或錯誤訊息反向搜尋，再沿著 import、呼叫與資料型別縮小範圍。

## 修改前先找規範

檢查 `AGENTS.md`、README、測試設定、格式化規則與既有元件，避免另造一套結構。
